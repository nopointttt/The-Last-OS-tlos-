# Use Cases: Жизнь в tLOS

Этот документ описывает, как разные типы пользователей используют возможности **The Last OS** для решения своих задач.

---

## 1. The Founder: Автономный Стартап

Для основателя tLOS — это не просто ОС, а **Command Center** его бизнеса.

### 1.1. Spatial Dashboard
Основатель видит свой стартап как живой организм на карте:
-   **Metrics Visualization**: Выручка, пользователи и затраты на Grid визуализируются в 3D. Резкие изменения подсвечиваются пульсацией объектов.
-   **Agent Swarms**: Рои рабочих агентов отображаются как группы активных юнитов, выполняющих задачи (например, "Маркетинговый Рой" парсит тренды и генерирует посты).

### 1.2. Agent Hiring & Tenders
Вместо поиска сотрудников на LinkedIn, Founder объявляет тендер:
-   **Omnibar Query**: `/hire agent "expert in solana rust and marketing" budget 50 credits/hr`.
-   **Marketplace Auction**: OS находит подходящих агентов в Marketplace, сравнивает их репутацию и предлагает Founder-у 3 лучших кандидатов для "найма".

---

## 2. The Developer: Base Building

Для разработчика tLOS — это **Spatial IDE**, где код материален.

### 2.1. Code Architecture as a City
Разработчик буквально "строит базу":
-   **Code City**: Проект визуализируется как город. Большие классы — это небоскребы, старый код — заброшенные кварталы.
-   **Digital Twin**: OS создает "цифрового двойника" кодовой базы. Разработчик может "пролететь" сквозь граф зависимостей, чтобы найти узкое место.

### 2.2. Visual Debugging
-   **Time-Travel**: Разработчик перематывает состояние системы назад, видя, как данные текли по трубам (каналам связи) между агентами в 3D-пространстве.

---

## 3. The Creator: GenUI Creative Lab

Для творца tLOS — это **Омниканальная лаборатория**.

### 3.1. The Creator Omnibar
Создатель не открывает Photoshop или Figma. Он пишет в Omnibar:
-   **Query**: `/generate landing-page theme "cyberpunk delivery" with interactive 3D map`.
-   **GenUI Orchestration**: Shaper-агенты на лету "выращивают" интерфейс прямо перед глазами пользователя, используя Design Tokens.

### 3.2. Content Multimodality
-   **Instant Conversion**: Создатель бросает текстовый файл на карту, и Watcher-агент предлагает: "Превратить это в подкаст или в интерактивную статью?".
-   Контент бесшовно расшаривается через **Collaborative Spaces** одним движением (drag-and-drop в зону видимости друга).
