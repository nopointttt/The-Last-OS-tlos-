# Semantic Memory: System Definitions

## 1. Сущности
- **Space (Холст)**: Виртуальное 2D пространство. Бесконечное.
- **Dynamic Component**: Контейнер для любого контента (окно).
- **Omnibar**: Центральная точка ввода. Всегда `Pinned`.
- **Universal UI Protocol (UUP)**: Стандарт JSON-описания интерфейса для динамической сборки.
- **Assembler Actor**: Актор-оркестратор, который собирает UI из данных других акторов.
- **Intent Resolver**: Механизм маппинга пользовательского ввода в системные "намерения" (Intents).
- **Lattice Assembly**: Режим работы, при котором Shell является чистым рендерером, а логика UI живет в Lattice.

## 2. Протокол (wRPC & Lattice)
- **Lattice (Латица)**: Децентрализованная сеть Актёров.
- **wRPC**: Высокопроизводительный протокол вызова удаленных процедур для Латицы.
- **WIT**: WebAssembly Interface Type, определяющий контракты между модулями.
- **Zero-Web2**: Архитектурный стандарт, запрещающий HTTP/REST внутри системы.

## 3. Снаппинг и Докинг
- **Ghost Preview**: Синяя пунктирная рамка, показывающая, куда "прилипнет" окно.
- **Docked**: Окно привязано к другому элементу (логически).

## 4. Глоссарий
- **Digital Citadel**: Модель безопасности ОС.
- **AI Firewall**: Прослойка фильтрации входных данных.
- **Sovereign Toolchain**: Использование Zig и Rust для независимости.
- **Quadtree Index**: Пространственная структура данных в Spatial Actor.
- **libcrux**: Библиотека с формально доказанной безопасностью (HACL*).
